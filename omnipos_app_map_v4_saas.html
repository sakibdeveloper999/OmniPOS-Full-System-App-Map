System Architecture
Three-layer MERN stack with Electron desktop app, React Native mobile, and a web-based QR receipt viewer. All communication goes through the REST API with Socket.io for real-time events.

Real-Time Events — Socket.io
📡 Socket.io Event Bus — Rooms are scoped per storeId
Clients join their store room on connect: socket.emit('join:store', storeId). All events are broadcast to that room only.
order:new
Fired after every completed checkout. Payload: orderId, orderNumber, total, cashier name, timestamp.
stock:updated
Fired on every stock change (sale, purchase, adjust, transfer). Payload: productId, newStock, delta, reason.
purchase:new
Fired when a new purchase from supplier is created. Payload: purchaseId, supplierId, total.
shift:opened
Broadcast when cashier opens a shift. Payload: shiftId, cashierName, openingCash.
shift:closed
Broadcast when shift closes. Payload: shiftId, totalSales, cashDifference, status.
lowstock:alert
Fired when stock drops at or below lowStockAlert threshold after a sale. Payload: productId, name, stock.

System Flows
Subscription Status Behavior
🟢 Active
Subscription is current and within expiry date. Full access to all plan features and limits. Normal operation for all users in tenant.
tenant.subscriptionStatus === 'active' && new Date() < tenant.expireAt
🟡 Expired
Subscription period ended. Features blocked at middleware level. Read-only access may be granted. Owner receives renewal prompt on login.
tenant.subscriptionStatus === 'expired' || new Date() >= tenant.expireAt
🔴 Suspended
Account manually suspended by SaaS Admin. Full lockout — no login possible. Used for payment disputes, ToS violations, or fraud prevention.
tenant.subscriptionStatus === 'suspended' || tenant.isActive === false
Middleware Pipeline (Every API Request)
1
JWT Decode
Extract userId, tenantId, role from Bearer token. Reject if missing, expired, or invalid signature.
const { userId, tenantId, role } = jwt.verify(token, SECRET)
↓
2
Tenant Load
Fetch tenant from DB. Attach to req. Check isActive = true and isActive tenant exists.
req.tenant = await Tenant.findById(tenantId)
↓
3
Subscription Check
Block request if subscription expired or account suspended. Return 403 with renewal message.
if (now > tenant.expireAt) return res.status(403).json({ error: 'Subscription expired' })
↓
4
Feature Check
Verify the plan includes the required feature for this route. Deny if feature not in plan.
if (!plan.features.includes('reports')) return res.status(403).json({ error: 'Upgrade plan' })
↓
5
Role Check
Verify role permission for this specific route. Cashier cannot access reports even on Pro plan.
if (!allowedRoles.includes(req.user.role)) return res.status(403)
↓
6
Tenant Isolation
ALL database queries must include tenantId filter. Never allow cross-tenant data leakage. Critical security layer.
Model.find({ tenantId: req.tenant._id }) // ALWAYS
↓
7
Limit Check
For create operations, verify usage is within plan limits. Block if at or over limit.
if (userCount >= plan.limits.users) return res.status(403).json({ error: 'User limit reached' })
↓
8
Controller → Response
All checks passed. Controller runs, data returned. All data scoped to tenant — zero cross-tenant risk.
res.json({ data: tenantScopedResults })

Security Rules (Never Break These)
✅ ALWAYS DO
Hash all passwords with bcrypt (salt 12)
Set JWT expiry on every token
Include tenantId in ALL Model.find() queries
Validate JWT before processing any request
Check subscription status on every protected route
Check feature access before executing business logic
Verify role permission per route
Rate limit login endpoint (brute force prevention)
❌ NEVER DO
Allow direct DB access without token
Trust tenantId from client request body
Run queries without tenantId filter
Return data from another tenant's query
Skip subscription check on any route
Expose cost prices to cashier role
Allow expired tenants to write data
Store plain-text passwords anywhere
