Atomic Transaction Flows
🛒 Checkout Flow
1
Scan barcode → GET /products/barcode/:code
2
Build cart in frontend (no API)
3
POST /orders — begin MongoDB transaction
4
Validate stock available per item
5
adjustStock(-qty) × N items + StockLog × N
6
Save Order document
7
Commit transaction (or abort all)
8
Emit socket event → store room
9
Generate HMAC-signed QR token
10
Electron prints receipt silently
🛍️ Purchase Flow
1
POST /purchases — begin transaction
2
Validate supplier belongs to store
3
adjustStock(+qty) × N items + StockLog × N
4
Save Purchase document
5
Supplier.currentBalance += balanceDue
6
Commit transaction
🔀 Stock Transfer Flow
1
POST /stock-transfers — deduct source
2
Status → "in_transit"
3
Destination PUT /:id/receive
4
adjustStock(+qty) at dest store
5
Status → "received"
🔙 Sales Return Flow
1
POST /sales-returns — begin transaction
2
adjustStock(+qty) × N items (restore)
3
Save SalesReturn document
4a
refundMethod=cash → pay out counter
4b
refundMethod=wallet → customer.walletBalance += amt
4c
refundMethod=points → loyaltyPoints += floor(amt)
5
Commit transaction
🔲 QR Receipt Verification
1
Customer scans QR code
2
GET /orders/receipt/:qrToken (public)
3
Base64 decode → orderId:timestamp:hmac
4
HMAC-SHA256 verify (timing-safe)
5
Check token age ≤ 30 days
6
Fetch order (strip costPrice)
7
Return { order, status: "valid" | "tampered" }

Security Stack
JWT access tokens (8h) + refresh tokens (30d). Silent re-auth on expiry.
bcrypt (salt 12) password hashing. Never stored or returned in plaintext.
Rate limiting: 20 req/15min on auth. 300 req/min on all API routes.
Joi schema validation on every POST/PUT. Unknown fields stripped automatically.
Helmet.js — sets X-Frame-Options, CSP, HSTS, X-Content-Type headers.
CORS restricted to allowed origins in production (env config).
costPrice stripped from all Cashier-role responses at controller level.
QR tokens: HMAC-SHA256 signed, 30-day TTL, timing-safe tamper detection.
MongoDB transactions ensure stock mutations are always atomic — no partial updates.
